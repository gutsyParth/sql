A relational database represents a collection of related (two-dimensional) tables.

To retrieve data from a SQL database, we need to write SELECT statements, which are often colloquially refered to as queries.

Select query for a specific columns
SELECT column, another_column, …
FROM mytable;

Select query for all columns
SELECT * 
FROM mytable;

In order to filter certain results from being returned, we need to use a WHERE clause in the query.

Select query with constraints
SELECT column, another_column, …
FROM mytable
WHERE condition
    AND/OR another_condition
    AND/OR …;

Quick Revision
Task	SQL
Title only	SELECT Title FROM Movies;
Director only	SELECT Director FROM Movies;
Title + Director	SELECT Title, Director FROM Movies;
Title + Year	SELECT Title, Year FROM Movies;
All columns	SELECT * FROM Movies;

Rule:

SELECT column_name → fetch specific columns.
SELECT col1, col2 → fetch multiple columns.
SELECT * → fetch all columns from the table.

4. Find the first 5 Pixar movies and their release year
SELECT Title, Year
FROM Movies
ORDER BY Year
LIMIT 5;

Output:

Title	Year
Toy Story	1995
A Bug's Life	1998
Toy Story 2	1999
Monsters, Inc.	2001
Finding Nemo	2003

LIMIT 5 returns only the first 5 rows after sorting by release year in ascending order.

4. Find all the WALL-* movies

The * means "anything after WALL-". In SQL, % is the wildcard character.

SELECT *
FROM Movies
WHERE Title LIKE 'WALL-%';

Output:

WALL-E

1. List all directors of Pixar movies (alphabetically), without duplicates

Use:

DISTINCT → removes duplicates
ORDER BY → sorts results
SELECT DISTINCT Director
FROM Movies
ORDER BY Director ASC;

Output:

Andrew Stanton
Dan Scanlon
John Lasseter
Lee Unkrich
Pete Docter
2. List the last four Pixar movies released (ordered from most recent to least)

Use:

ORDER BY Year DESC → newest first
LIMIT 4 → return only 4 rows
SELECT *
FROM Movies
ORDER BY Year DESC
LIMIT 4;

Output:

Title	Year
Monsters University	2013
Cars 2	2011
Toy Story 3	2010
Up	2009
3. List the first five Pixar movies sorted alphabetically

Sort by title and take the first 5 rows.

SELECT *
FROM Movies
ORDER BY Title ASC
LIMIT 5;

Output:

Title
Cars
Cars 2
Finding Nemo
Monsters University
Toy Story
4. List the next five Pixar movies sorted alphabetically

Skip the first 5 rows and return the next 5.

SELECT *
FROM Movies
ORDER BY Title ASC
LIMIT 5 OFFSET 5;

Output:

Title
Toy Story 3
Up

(Only two rows remain in this dataset after skipping the first five.)

Quick Revision
DISTINCT

Removes duplicate values.

SELECT DISTINCT Director
FROM Movies;
ORDER BY

Ascending (A → Z, Small → Large)

SELECT *
FROM Movies
ORDER BY Title ASC;

Descending (Z → A, Large → Small)

SELECT *
FROM Movies
ORDER BY Year DESC;
LIMIT

Return only a certain number of rows.

SELECT *
FROM Movies
LIMIT 4;
OFFSET

Skip a number of rows.

SELECT *
FROM Movies
LIMIT 5 OFFSET 5;

Meaning:

Sort the data.
Skip the first 5 rows.
Return the next 5 rows.

A common pattern for pagination is:

SELECT *
FROM Movies
ORDER BY Title
LIMIT 5 OFFSET 0;   -- Page 1

SELECT *
FROM Movies
ORDER BY Title
LIMIT 5 OFFSET 5;   -- Page 2

SELECT *
FROM Movies
ORDER BY Title
LIMIT 5 OFFSET 10;  -- Page 3



1. List all the Canadian cities and their populations
SELECT City, Population
FROM North_american_cities
WHERE Country = 'Canada';
2. Order all the cities in the United States by their latitude from north to south

Since latitude decreases as we move south, sort by latitude in descending order.

SELECT City
FROM North_american_cities
WHERE Country = 'United States'
ORDER BY Latitude DESC;

Or, if you want to see the latitude too:

SELECT City, Latitude
FROM North_american_cities
WHERE Country = 'United States'
ORDER BY Latitude DESC;
3. List all the cities west of Chicago, ordered from west to east

Chicago's longitude is approximately -87.629798.

Cities west of Chicago have a longitude less than -87.629798 (more negative).

SELECT City
FROM North_american_cities
WHERE Longitude < -87.629798
ORDER BY Longitude ASC;

Or:

SELECT City, Longitude
FROM North_american_cities
WHERE Longitude < -87.629798
ORDER BY Longitude ASC;

Why ASC?

More negative = farther west
Less negative = farther east

Example:

-112  (west)
-103
 -99
 -95
 -87  (east)

5. List the third and fourth largest cities (by population) in the United States and their population

Steps:

Consider only U.S. cities.
Sort by population descending.
Skip the first two largest cities.
Return the next two.
SELECT City, Population
FROM North_american_cities
WHERE Country = 'United States'
ORDER BY Population DESC
LIMIT 2 OFFSET 2;

Explanation

Suppose the U.S. cities sorted by population are:

Rank	City
1	New York
2	Los Angeles
3	Chicago
4	Houston
5	Philadelphia

OFFSET 2 skips:

New York
Los Angeles

and LIMIT 2 returns:

Chicago
Houston
Concepts Used
Clause	Purpose
WHERE	Filter rows
ORDER BY ASC	Small → Large / A → Z
ORDER BY DESC	Large → Small / Z → A
LIMIT n	Return first n rows
OFFSET n	Skip first n rows
LIMIT n OFFSET m	Skip m rows, then return n rows
Common Pattern
SELECT columns
FROM table
WHERE condition
ORDER BY column DESC
LIMIT n OFFSET m;

This pattern appears very frequently in interviews and real-world SQL queries.
